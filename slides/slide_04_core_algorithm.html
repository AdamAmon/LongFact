Core Algorithm 核心模型与算法：DCE 双通道证据检索

检索哲学：更精确的证据 > 更多的证据

DCE（Dual-Channel Evidence Retrieval with Entailment-Guided Re-Ranking）包含四项原创机制，均为零额外模型开销设计。

01
双通道检索融合
Dual-Channel Fusion
语义嵌入（SBERT + FAISS）+ 关键词 BM25 独立检索，加权合并去重（0.6 / 0.4），弥补词汇语义鸿沟。
semantic + keyword
02
轻量级重排序
Lightweight Re-Ranking
n-gram Jaccard 重叠（60%）+ 实体命中 + 位置连贯性，三项启发式打分，无需额外模型推理。
zero additional cost
03
自适应 Top-K
Adaptive Top-K
短句 k=3 / 中等 k=5 / 长句 k=7，基于句子词数动态调整，减少无效 NLI 调用。
sentence-length aware
04
条件证据扩展
Conditional Expansion
NLI 置信度 < 0.6 时激活，以 top-3 锚点在 FAISS 索引中找最近邻段落，按需扩展候选池。
on-demand activation

# DCE 增强版 query：双通道 → 重排序 → top-k
def query(self, text, top_k=5):
    adaptive_k = self.adaptive_top_k(text, base_k=top_k)

    # 通道 1: 双通道检索（语义 + 关键词）
    candidates = self._dual_channel_search(text, adaptive_k)

    # 通道 2: 轻量级重排序
    ranked = self._rerank(text, candidates)

    # 记录供条件扩展使用
    self._last_top_indices = [idx for idx, _ in ranked[:adaptive_k]]

    return ranked[:adaptive_k]
