Innovation 项目创新点

四项原创贡献 · 算法设计完全自主

01
INNOVATION · Algorithm

DCE 双通道证据检索

原创设计：语义 + 关键词双通道融合，配合零额外模型开销的轻量级重排序（n-gram Jaccard + 实体命中 + 位置一致性），与自适应 Top-K 和条件证据扩展协同工作。4 项机制均为 原创设计，未借鉴已有论文方法。
vs 现有方法 标准检索：单一语义通道 + 固定 Top-K，噪声证据 → 假阳性 ENTAILMENT → DCE 减少假阳性 6.16pp
02
INNOVATION · Engineering

完整的 Generate–Detect–Correct–Evaluate 管道

从分块摘要 → 证据检索 → 句子级 NLI → 局部纠错 → 双重评估，每个模块可独立替换和实验。支持 fp16 / 8-bit / CPU 回退多精度配置，在 8GB VRAM 上稳定运行 500 样本实验。
vs 现有方法 多数研究仅覆盖单一环节，缺乏端到端可复现实验框架 → 本项目提供完整代码 + 16 项测试 + 分阶段耗时统计
03
INNOVATION · Analysis

ROUGE 与事实一致性正交的实证证明

通过 500 样本定量实验 + 20 个案例深度分析，系统证明了：自动纠错可将支持率提升 +10pp，而 ROUGE 变化 < 0.002。从微观测例（DCE Case 305：支持率 +0.3052 而 ROUGE 几乎不变）到宏观统计，完整揭示了 ROUGE 作为事实评估指标的根本缺陷。
vs 现有方法 已有论文讨论了 ROUGE 的不足，但缺乏大规模+细粒度案例的双重实证
04
INNOVATION · Methodology

长度分层分析 + 纠错案例双策略对比

针对 GovReport 样本全落在 16+ 桶的问题，设计百分位分桶策略（每桶 125 个样本），揭示四个规律：支持率随长度递减、校正增益随长度递增、ROUGE 随长度锐减、NLI 分布长度稳定。同时按 Top-5 改善 / Top-5 恶化 选取案例，Baseline 和 DCE 各 10 例进行交叉分析。
vs 现有方法 常规等距分桶在长尾数据中失效 → 百分位分桶 + 双策略案例交叉分析
